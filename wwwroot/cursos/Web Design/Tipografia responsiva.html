Tipografia responsiva

O último tipo de elemento ao qual devemos aplicar técnicas de responsividade, e talvez o mais importante de todos é o texto. Você já sabe que em vez de definir um tamanho específico para cada elemento de texto, como um parágrafo ou título, o que deve ser feito é definir um tamanho base, e a partir daí utilizar em ou rem como unidade de medida, certo?

<style>
html {
  font-size: 1rem;
}
p {  font-size: 2em;  }
h1 {  font-size: 5.5em;  }
h6 {  font-size: 3em;  }
</style>
<h1>Isto é um título?</h1>
<h2>Isto é um subtítulo?</h2>
<p>Algum texto em alguma parágrafo que você escreveu sem perceber...</p>

Percebeu como essa abordagem facilita o redimensionamento da fonte em diversas ocasiões? Pois basta alterar um tamanho de fonte, nesse caso, na raiz do html, para provocar um efeito cascata atualizando as demais.

A abordagem acima pode ser muito útil ao tratar a responsividade da página, seja com grades, com caixas flex, ou com consulta de mídia, por exemplo:

<h1>Isto é um título?</h1>
<p>Algum texto em algum parágrafo que você escreveu sem perceber...</p>
<style>
html {  font-size: 1vw;  }
p {  font-size: 2em;  }
h1 {  font-size: 3em;  }


@media (max-width: 800px) {


  html {


    font-size: 2vw;


  }


}


</style>

No exemplo acima a consulta de mídia é utilizada para dobrar o tamanho da fonte caso o tamanho da tela seja menor que 800px.

Você estranhou o valor 2vw, certo? Não entraremos muito a fundo nisto, apenas acrescentei-o aqui a título de demonstração. A unidade vw (viewport width) representa 1% da largura da viewport, ou seja, da janela de exibição, apesar de garantir maior fidelidade à tela em diferentes dispositivos, pode terminar causando efeito contrário e diminuindo a acessibilidade da página, pois 1% de tela, sempre será 1% de tela, fazendo com que o usuário não possa ampliar ou reduzir o texto com o zoom. Entretanto, no exemplo acima, ao utilizarmos consulta de mídia, garantimos que o texto dobre de tamanho conforme a ampliação de zoom faça a resolução da tela ficar menor que 800px (o zoom amplia os pixels fazendo com que menos caibam na tela). Mesmo assim pode não ser uma experiência satisfatória para o usuário. Mas essa abordagem funciona de forma perfeita com outras unidades de medida como pixels, porcentagem etc.

As unidades vw (viewport width) e vh (viewport height) ainda cabem muito bem no dimensionamento de outros elementos, tendo em mente que 1vh representa 1% da altura da tela, e 1vw, 1% da largura da tela.
