Layout de página

Ao construir um site devemos considerar alguns conceitos, o primeiro é o layout da página, que pode variar de acordo com o propósito do site, as informações exibidas, a interação do usuário, a clareza de informações, e como o site se adapta em diferentes telas. O layout pode até mesmo variar de página para página num mesmo site, no entanto, o layout básico pode ser assim:

Neste exemplo, temos os elementos de layout mais básicos de uma página HTML, veremos como construir um site com cada uma desses elementos, com a única diferença de que nessa imagem definimos duas barras laterais para demonstrar as possíveis localizações, mas o uso de duas barras diminui drasticamente a quantidade de espaço destinado ao corpo.

Apesar de ser possível utilizar elementos div, ou outros elementos HTML para cria os componentes acima, como menu, cabeçalho, barras laterais e rodapé, o HTML possui elementos específicos que possuem a função semântica própria para sua finalidade, como você pode notar a seguir:

<menu>Meu menu</menu>
<header>Meu cabeçalho</header>
<aside>Minha barra lateral</aside>
<article>Meu artigo</article>
<footer>Meu rodapé</footer>

Estes elementos por si só não produzem o resultado que desejamos e funcionam como qualquer elemento div, apenas servindo de contêiner. Para que produzam o resultado esperado, devem, além de abrigar o conteúdo necessário como parágrafos, botões, divs, imagens etc., também devem ser estilizados de acordo.

Construir um layout de página elegante, eficiente e responsivo requer o conhecimento das técnicas mais modernas de design de interface e bastante trabalho. Felizmente o HTML e o CSS nos permitem, sem uso de Javascript, e sem necessidade de conceitos avançados de design de aplicativos e programação de gráficos e multimídia, alcançar estes resultados.

Um layout responsivo significa que a página se adapta a diferentes tamanhos de tela, como computador, tablet e PC, ou ao redimensionamento da janela do navegador. Uma página responsiva acessada de um celular tem o design de uma página móvel, acessada de um tablet tem o design de uma página para telas menores, e acessada do PC tem a aparência de uma tela capaz de exibir informações de forma eficiente em grandes telas.

Mas além disso, devemos incluir conceitos de acessibilidade, que significa que os elementos respondem de maneira esperada a diferentes tipos de interação, como por exemplo com o toque de tela, o mouse ou uma caneta. Para se ter uma ideia, ao pararmos o mouse num elemento ele pode mudar de estado, isso não acontecerá num celular pois a interação é feita com toque de tela, e ao tocar o elemento responderá como se tivesse sido clicado.

A acessibilidade diz ainda da experiência do usuário ao navegar, uma navegação eficiente, prazerosa e não frustrante, onde as informações são exibidas de maneira concisa e de fácil interpretação, e o usuário obtém as respostas esperadas de suas ações, no momento esperado.

Lembre-se sempre das informações acima! Para alcançar isso devemos trabalhar bem desde os elementos HTML, os estilos CSS, até o head do documento, que é o local onde inserimos as metatags, o SEO e importamos referências externas. Veremos tudo isso!
