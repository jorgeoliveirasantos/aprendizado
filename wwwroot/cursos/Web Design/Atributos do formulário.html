Atributos do formulário

Action

É o atributo action que definirá o que acontece com os dados inseridos no formulário após o usuário clicar no botão submit:

<form action="https://urlServer.com/api" method="post">
    <label >Nome</label>
    <input type="text">
    <input type="submit" value="Enviar">
</form>

O atributo action define a URL de um servidor com API capaz de receber uma solicitação HTTP, solicitações HTTP comuns são GET (obter), POST (criar), PUT (alterar) e DELETE (excluir). Para compreender melhor este assunto confira:

developer.mozilla.org/pt-BR/docs/Web/HTTP/Methods

Ou… aguarde os próximos cursos!

Note que definimos o método da requisição adicionando o atributo method="post". Portanto o valor passado no atributo action deve ser a URL válida de uma API no servidor apta a receber tais solicitações. O servidor processará os dados de acordo com a lógica nele definida e enviará uma resposta para sua página, essa resposta pode ser exibida na mesma página ou numa nova guia. Para exibi-la numa nova guia adicionamos o atributo target:

<form action="https://urlServer.com/api" target="_blank">

Caso você se esqueça de adicionar o atributo method em seu formulário, o envio dos dados será numa solicitação GET, e os dados do usuário estarão expostos. Nunca envie formulários em solicitações GET a menos que seja o único método aceito pelo servidor.

Esfrie sua cabeça! Não é preciso se preocupar com APIs, solicitações HTTP, e tudo isso por enquanto, pois isto faz parte do desenvolvimento de aplicações de backend que não fazem parte do escopo deste curso, mas caso fique curioso, confira o curso de Aplicações Web.

Autopreenchimento

A maioria dos navegadores contem recursos de autopreenchimento de formulários:

Este recurso vem desativado por padrão em formulários HTML, para ativá-lo basta adicionar o atributo autocomplete, assim:

<form autocomplete="on" action="https://urlServer.com/api">

Validação

Quando um usuário clica no botão submit do formulário, caso um campo esteja preenchido incorretamente, como um campo de e-mail por exemplo sem um sinal de @, o navegador acusará erro. Se não for necessário essa validação e você deseje que o usuário possa inserir qualquer valor nos campos do formulário, basta adicionar o atributo novalidate, não é necessário passar nenhum valor ao atributo:

<form action="https://urlServer.com/api" novalidate>

Não comece a utilizar validação apenas com estes conhecimentos, pois um usuário com conhecimento básico de desenvolvimento web poderá alterar estes parâmetros simplesmente editando o código fonte de sua página no navegador dele. A validação geralmente é feita no servidor e não na página web.

Evento onsubmit

Ao submetermos um formulário, um evento onsubmit é disparado no navegador, é possível utilizar este evento para, por exemplo, executar instruções Javascript, veremos mais sobre isso a frente, mas a maneira de fazê-lo é a seguinte:

<form onsubmit="alert('Formulário enviado!')">

Ou:

<form onsubmit="minhaFuncaoJS()">
    <input type="submit" value="Enviar">
</form>
<script>
function minhaFuncaoJS() {
  alert('Formulário enviado ao servidor!');
}
</script>

Note que diferente de outros atributos, os atributos de evento, como onsubmit ou onclick, recebem não apenas valores em texto, mas instruções Javascript.
