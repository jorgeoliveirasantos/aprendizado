Links

A navegação em documentos HTML pode ser feita de várias maneiras:

1. Por meio de uma âncora <a>.

2. Por meio do evento de clique de um botão <button>.

3. Por meio do evento de clique de qualquer elemento HTML.

Ressaltando que a navegação não precisa obrigatoriamente levar o usuário para outra página, mas pode ser usada para baixar um arquivo ou levar o usuário para outra posição na mesma página. Links podem ainda ser utilizados para inserir referências de arquivos externos dentro de um documento HTML.

Âncora

Para criar um link de navegação para outro arquivo HTML, utilizamos a tag <a>, e definimos o caminho no atributo href, da seguinte maneira:

<a href="meuDocumento2.html"><p>Clique-me!</p></a>

O texto “Clique-me!” será exibido como um link na página e ao ser clicado, encaminhará o usuário para o arquivo “meuDocumento2.html”.

Para que a nova página seja exibida numa nova guia, definimos um “target blank”, assim:

<a href="meuDocumento2.html" target="_blank">
  <p>Clique-me!</p>
</a>

O arquivo deve estar na mesma pasta que o arquivo onde o link foi criado. Veremos muito mais sobre navegação num capítulo específico.

Botão

A navegação por meio de botões é feita com o evento de clique, que deve receber uma instrução Javascript, assim:

<button onclick="window.location = 'meuDocumento2.html'">
  Clique-me!
</button>

Para que o link seja aberto numa nova guia, basta alterarmos a instrução para window.open():

<button onclick="window.open('meuDocumento2.html')">
  Clique-me!
</button>

Evento de clique em elementos

	O mesmo que fizemos com o botão pode ser feito com qualquer elemento HTML:

<h3 onclick="window.open('meuDocumento2.html')">
  Clique-me!
</h3>


          
 No entanto, mesmo que a navegação ocorra ao clicarmos, o comportamento não é de um link, pois ao passarmos
            o mouse por cima, nada acontece, isso porque é preciso definir estilos CSS. Veremos tudo isso mais adiante.
          

          
Âncora de página

          
 Uma âncora de página visa levar o usuário para determinada posição dentro da mesma página baseado na id de
            um elemento:

          
<a href="#meuParagrafo">Descer!</a>
<div style="height: 2500px;"></div>
<p id="meuParagrafo">Chegou!</p>


          
 Neste exemplo, uma div com uma altura de 2500px coloca o parágrafo no final da página e fora da tela,
            usando o atributo href da âncora com uma cerquilha # na frente do valor, encaminhamos o usuário para o
            elemento com a id especificada.

          
 Este mesmo processo pode ser feito para encaminhar o usuário para qualquer parte de outro arquivo HTML:
          

          
<a href="documento2.html#algumElemento">Clique-me!</a>


          
 Este exemplo encaminha o usuário para outro documento HTML especificando a posição na rolagem da página
            onde se encontra o elemento com a id algumElemento.

          
 Mas pode ser qualquer página da web:

          
<a href="https://developer.mozilla.org/en-US/docs/Learn#contact_us">
  Clique-me!
</a>


          
 Esta é uma URL válida e encaminha o usuário para a página do MDN (Mozilla Developers Network), em uma
            posição específica de acordo com a id de um elemento existente naquela página.

          
Caminhos de arquivos

          
 Ao referenciar caminhos de arquivos em links HTML temos várias opções, como, por exemplo, arquivos na web:
          

          
<img src="https://minhapagina.com/minhaimagem.png">


          
 O mesmo vale para redirecionar o usuário para uma página HTML:

          
<a href="https://minhapagina.com/minhapagina.html">
Navegar</a>


          
 Para referenciar arquivos dentro da pasta do seu próprio site, você pode utilizar Caminhos Relativos,
            assim:

          
<link href="minhaimagem.png">


          
 O exemplo acima acessará a imagem caso ela se encontre na mesma pasta em que a página está. Caso ela
            esteja uma pasta acima, podemos fazer assim:

          
<link href="../minhaimagem.png">


          
 No exemplo acima, a imagem está numa pasta um nível acima que a página de onde ela é chamada, para dois
            níveis acima fazemos o seguinte:

          
<link href="../../minhaimagem.png">


          
 Você pode subir quantos níveis for necessário e ainda acessar outras pastas que constem em determinado
            nível. Para acessar pastas em outros níveis:

          
<link href="../minhapasta/minhaimagem.png">


          
 Neste exemplo, a pasta “minhapasta” está um nível acima que o arquivo onde o link é chamado, e a imagem
            “minhaimagem.png” está dentro dela.

          
 Uma alternativa mais simples é utilizar Caminhos Absolutos:

          
<link href="/minhaimagem.png">


          
 Neste exemplo a imagem está na pasta raiz do site. A / no início referencia a pasta raiz, e depois dela
            podemos referenciar qualquer caminho dentro da raiz, por exemplo:

          
<link href="/minhapasta/subpasta/minhaimagem.png">


          
 Neste exemplo, minhapasta está na raiz do site. O uso de Caminho Absoluta para sites prontos para
            publicação é o ideal facilitando as referências, e geralmente a / aponta para a pasta raiz onde todo o site
            está, entretanto, durante o desenvolvimento, ela pode apontar para o disco rígido (C:\, E:\ etc).
            Estudaremos isso no momento certo.
