Conteúdo incorporado

Nem sempre será possível criar conteúdo dentro do HTML, e precisaremos inserir conteúdo de um arquivo externo, seja na pasta do projeto, seja de um link na internet, como por exemplo, um vídeo do Youtube, um PDF, etc. Chamamos esse tipo de conteúdo, de Conteúdo Incorporado, e para esse trabalho possuímos os elementos, <iframe>, <embed> e <object>, sendo o iframe mais recomendado para inserir documentos, e o embed para inserir mídia, estes dois são os de uso mais recomendado e que trarão o resultado esperado.

Documentos

Utilizamos um iframe para inserir documentos como páginas web e HTML, por exemplo:

<iframe src="minhaPasta/meuDocumento.html"></iframe>

O caminho do documento a ser exibido é passado no atributo src que define a origem. Ao inserir um iframe algumas propriedades padrão serão definidas como largura e altura mínimos e borda. Para alterar estas propriedades devemos especificar estes atributos, assim:

<iframe src="https://developer.mozilla.org/pt-BR/" 
width="100%" height="300px" frameborder=0></iframe>

Detalhe: Durante o desenvolvimento, enquanto exibimos nosso arquivo da pasta local, a maioria dos sites recusará a conexão por motivos de segurança, será necessário acessar o arquivo publicado num servidor para que o conteúdo do iframe seja exibido corretamente.

Para desativar a rolagem do conteúdo do quadro, alteramos a propriedade scrolling: Sem barra de rolagem (impede a rolagem do conteúdo): 

<iframe src="www.jorgesouza.com" scrolling="no"></iframe>

Barra de rolagem sempre exibida:

<iframe src="www.jorgesouza.com" scrolling="yes"></iframe>

Barra de rolagem sempre automática (apenas quando o conteúdo excede a altura do quadro):

<iframe src="www.jorgesouza.com" scrolling="auto"></iframe>

Caso o quadro exiba um conteúdo que solicite tela cheia, é necessário permitir esse acesso definindo allowfullscreen como true:

<iframe src="www.jorgesouza.com" 
allowfullscreen="true"></iframe>

Sandbox

O atributo sandbox define uma quantidade de propriedades que visam permitir o conteúdo do quadro ter certos comportamentos, um exemplo é:

<iframe src="www.jorgesouza.com"
sandbox="allow-popups"></iframe>

Neste exemplo, estamos permitindo que o conteúdo dentro do quadro exiba caixas pop-up, utilizamos allow-forms: para permitir o envio de formulários e allow-scripts: para permitir que o conteúdo execute scripts (essa propriedade pode criar brechas de segurança sérias em sua página).

Vídeo

Podemos inserir mídia num quadro dentro de um documento HTML utilizando tanto <iframe> como <embed>, na realidade, a maioria dos sites que permitem incorporação de seu conteúdo gera tags iframe, um exemplo é o Youtube. Vamos pegar este exemplo, clique com o botão direito sobre um vídeo em execução no Youtube e copie o código de incorporação, deve ser semelhante a este:

<iframe width="727" height="409"
src="https://www.youtube.com/embed/yICVNta8jMU"
title="Introducing Project Volterra (Satya Nadella 2022 Build Keynote)"
frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
allowfullscreen></iframe>

Perceba que o Youtube já fornece um frame configurado, sem borda, com a propriedade title, com largura e altura etc. O uso do atributo allow permite ainda definir quais recursos do Youtube serão permitidos em nossa incorporação, por exemplo, se removermos o valor picture-in-picture, o frame não poderá acessar esse recurso do navegador. O mesmo ocorre se removermos o atributo allowfullscreen, o botão de tela cheia do Youtube será inativado.

Podemo utilizar o mesmo código gerado pelo site alvo, ou apenas copiar o link da mídia, e criar um conteúdo incorporado com um elemento embed. No caso de utilizar o código, devemos remover os atributos não suportados para que não haja confusão:

<embed width="727" height="409"
src="https://www.youtube.com/embed/yICVNta8jMU"
title="Introducing Project Volterra (Satya Nadella 2022 Build Keynote)">
</embed>

Perceba que a diferença entre utilizar um ou outro modo de incorporação é apenas questão de escolha, sendo que na maioria dos casos um iframe é suficiente.

Áudio

Para adicionar conteúdo de áudio o processo é o mesmo, ficando a critério do navegador criar o player para reproduzir o conteúdo:

<embed src="Asher Fulero - Kiss The Heavens.mp3" type="audio/mpeg">

Você notou o atributo type neste código? Este atributo não é necessário pois os navegadores costumam reconhecer o tipo de arquivo, mas pode ser útil caso o servidor reporte ao navegador o tipo incorreto causando erros na reprodução. O atributo type recebe como valor uma string de acordo com a especificação de tipos de arquivo (MIME) que pode ser encontrada em: https://developer.mozilla.org/pt-BR/docs/Web/HTTP/Basics_of_HTTP/MIME_types/Common_types

Controles HTML

Você pode ainda criar controles para áudio e para vídeos com as tags <audio> e <video>:

<audio src="minhaMusica.mp3" controls></audio>
<video src="meuVideo.mp4" controls></video>

O atributo controls é que criará os controles para reprodução do conteúdo.

Apesar de raro atualmente, caso o navegador utilizado pelo usuário não ofereça suporte à incorporação de conteúdo, podemos definir entre as tags do elemento um texto a ser exibido alternativamente:

<audio src="minhaMusica.mp3" controls>
  Sinto muito! Este navegador não oferece suporte ao conteúdo incorporado.
</audio>

Os elementos audio e video permitem definir o comportamento da mídia com alguns atributos:

autoplay: Inicia a reprodução automaticamente.

muted: Inicia com o áudio do player mudo.

loop: Reinicia a reprodução quando chegar ao fim.

Imagens

Imagens são inseridas com o elemento <img> que é muito simples de ser utilizado:

<img src="E:\Downloads\BingWallpaper.jpg" width="500px" height="auto"
alt="texto alternativo da imagem" title="Essa é uma imagem!">

Ao especificarmos atributos width e height simultaneamente a imagem pode perder sua proporção, para evitar isso, podemos omitir um dos atributos ou defini-lo como auto. O atributo alt é utilizado para inserir uma mensagem, geralmente uma descrição da imagem que será exibida em seu lugar enquanto a imagem é carregada (geralmente quando a largura de banda da internet do usuário é baixa).

Notas sobre os formatos

Para manter a qualidade de seus arquivos e reduzir o tamanho, o que reduz o tempo de carregamento da página, utilize formatos modernos em vez de clássicos como .png, .jpg, .mp3, .mp4 etc. Os formatos mais modernos disponíveis são .webp (imagem), e .webm (audio/vídeo). A maioria dos editores já fornece suporte à exportação nesses formatos, e também a maioria dos sites onde fazemos download também oferecem seus arquivos nesses formatos. Utilizar conversores, principalmente online não é a mesma coisa, pode até dar certo, mas pode ter resultados inesperados ou não causar muito impacto no tamanho final do arquivo.
